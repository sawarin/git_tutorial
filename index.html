Gitチュートリアル

git clone <リポジトリ名>

gitリモートリポジトリ(github)のコピーをローカルリポジトリに作成する
→ git clone https://github.com/sawarin/git_tutorial.git

git add <ファイル名・ディレクトリ名・.>

ワークツリーの変更をステージに追加する
→ git add index.html

git commit

ステージの変更をローカルリポジトリに記録する
→git commit

git status

現在の状況を確認する
→git status

git diff

ワークツリーとステージとの差分(git addする前の変更分)
→git diff

git log

変更履歴を確認する
→git log --online
→git log -p index.html
→git log -n 1

git rm

ローカルリポジトリ及びワークツリーからファイルの削除を記録する
→git rm index.html ※戻すときはgit reset HEAD/ git checkout

ローカルリポジトリからのみファイルの削除を記録する
→git rm --cached index.html ※戻すときはgit reset HEADのみ。 ワークツリーには残っているのでcheckoutは不要

git mv

ファイル名を変更・移動してステージングに記録する。mv→git rm→git addしているのと同じでローカルリポジトリには記録されない
→git mv index.html index2.html

git remote add <リモート名> <リモートURL>

リモートリポジトリ(github)をローカルリポジトリする前(pushする前)の準備として、ローカルに新規追加する。パーソナルトークンの登録も必要
→git remote add origin https://github.com/sawarin/git_tutorial.git
→git remote add bak https://github.com/sawarin/git_tutorial_bak.git

git push <リモート名> <ブランチ名>

ローカルリポジトリの内容をリモートリポジトリに送る。-uは次回以降origin masterをpushする際にgit pushだけでOKになるオプション
→git push -u origin master
→git push origin master
→git push -u bak master
→git push bak master

git checkout -- <ファイル名・ディレクトリ名・.>

ワークツリーに追加したけど変更を取り消す。ワークツリーをステージングの内容に書き換える
→git checkout -- index.html

git reset HEAD <ファイル名・ディレクトリ名・.>

　

ステージに追加したけど変更を取り消す。ステージングをローカルリポジトリの内容に書き換える。ワークツリーの情報は変わらないため、
→git reset HEAD index.html

git commit --amend

　

直前(1つ前)にcommitした内容をやり直す（修正する)。リモートリポジトリにpushした後に実施するのは絶対NG！
→git commit --amend

git remote

　

設定しているリモートリポジトリの情報を表示する
→git remote -v

git fetch <リモート名>

　

リモートリポジトリからローカルリポジトリに情報を取得（フェッチ）する。ワークツリーに反映されない、するためにはgit mergeが必要
→git fetch origin　※ワークツリーに反映するにはgit mergeが必要

git branch

　

ブランチの一覧を表示する。リモートでフェッチしたリポジトリも一覧に表示される。
→git branch -a　※「*」がついているところが今いるところ

git checkout

　

ワークツリーの内容を切り替える
→git checkout remotes/origin/master
→git checkout master

git merge

　

git fetchで取得したローカルリポジトリの内容をワークツリーに反映する。
→git merge origin/master

git pull <リモート名> <ブランチ名>

　

「git fetch」と「git mmerge」が1回で実行できるコマンド。ただし、今いるブランチにmergeされちゃうので注意
→git pull origin master
